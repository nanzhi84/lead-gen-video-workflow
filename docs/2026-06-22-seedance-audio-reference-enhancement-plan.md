**结论:它是「口播实拍镜头 + 空镜素材混剪 + 自动字幕 + BGM」的智能成片,不是 Seedance 那种"文字凭空生成画面"。** 由此:
1. **口型问题不存在/已被绕过**:口播是真人镜头(或带口型的生成),且大量用空镜盖在画外音上——所以 115 字快语速塞 15s 完全 OK。我之前"15s 塞不下、口型漂移"的担忧**作废**。
2. **画幅是 3:4(720×960),不是 9:16**——我现在硬编码 9:16 不对,要做成可配/默认跟老板一致。
3. **画面是"真实素材",不是 AI 生成的**——这引出下面这个**必须你拍板的岔路**。

**岔路:这条视频画面到底怎么来的?决定我们怎么实现**

| 路线 | 画面怎么来 | 像不像老板这条 | 对应我们系统 |
|---|---|---|---|
| **A 实拍混剪** | 用老板**上传的真实素材**(老板娘口播视频 + 店内空镜)按脚本**混剪** + 自动字幕 + BGM | **一模一样**(真店真人) | 接近现有 broll/数字人链路(画外音/口播+空镜+字幕+BGM),把素材改成 @ 指定 |

> 你这次要的 **"AI素材 tab"**(见 §9)无论走 A 还是 B 都要做——它就是"按 case 上传图/视频素材、之后 @ 引用"的素材管理,A 用它当混剪素材、B 用它当生成参考。所以这块可以先确定下来。

## 0. 老板的 prompt 逐行对照

```
请直出一条抖音信息流广告视频，配 BGM。        ← 指令(竖屏/广告/BGM)
这是口播脚本。{在陵水海风小镇…来逛逛。}        ← 要被【念出来】的台词(约115字/5句)
老板娘出镜@98edf3bd-da31-449c-8d73-647de363fec1 ← 持久化素材id(老板娘的图/视频)
```

Seedance 2.0 **完全能做这件事**(已联网核实):把台词用引号写进 prompt → 画面人物会念出来并对口型(支持普通话);参考图/视频锁定人物长相;BGM 是原生音轨。`@<id>` 就是一条已上传的持久素材。

## 1. 关键发现一:持久复用「已经成立」(回答点3)

**"上传一次、后续按 @id 复用 N 次"系统现在就支持,无需每次重传。** 证据:

- 上传链路:`/api/uploads/prepare` → PUT 文件 → `/api/uploads/complete` 自动建 `MediaAssetRecord`(稳定 id 存 DB)+ source artifact + 缩略图(`apps/api/services/uploads.py`)。
- 素材作用域:`MediaAssetRecord.case_id` 是 **可空**(`media.py:85-108` / `database.py:258-273`)→ 素材可绑某 case,也可全局(case_id=NULL),**天然支持跨 case 复用**(生成时 `asset.case_id in {None, request.case_id}`)。
- 既有复用先例:数字人 `portrait.specific_template_id` / `template_sequence_ids`、封面 `cover.reference_asset_id`(`jobs.py`)——全是"存一次、按 id 引用 N 次"。我加的 `reference_asset_ids` 就是同一套。
- 我的 `SeedanceReferencePicker` 本就从 `api.mediaAssets.list({case_id})`(持久库)读取,**不是当场上传**。
- `case_reference_assets`(codex 的角色绑定表)确认不存在,也不需要——按 asset_id 引用即可。

**所以点3的核心诉求已满足。** 差的是三个补丁(下面方案覆盖):
1. **视频参考**:选图器现在只列图片(`IMAGE_KINDS`),老板娘若是**视频**素材选不到。
2. **@mention 输入交互**:现在是网格多选(功能等价),没有"输入框里打 @ 弹出素材"的富文本交互。
3. **跨 case 复用的可见性**:技术上支持,但选图器按 `case_id` 过滤,只显示当前 case 的素材;要在多个 case 复用同一条"老板娘",需让该素材全局可见或选图器也列全局/用户级素材。

## 2. 关键发现二:⚠️「15s + 全脚本 + 对口型」三者不可兼得(最重要)

Seedance 2.0 的对白口型有硬限制(已核实):
- **15s 片子,台词建议 ≤ ~20 词(英文)**;中文/方言**口型漂移更快**,clip 后半段"嘴和声音会对不上"。
- 老板的口播脚本是 **5 句、约 115 字**,正常语速念完要 **30~40 秒**,**塞不进 15s**——硬塞要么语速飙到失真,要么口型从中段开始全乱。

**这是"15s + 不后包装 + 真人口播"三个约束撞出来的物理矛盾,必须做产品取舍。** 三个选项:

| 选项 | 做法 | 代价 |
|---|---|---|
| **A. 砍台词(推荐)** | 把口播压成一句钩子(约 25~40 字,≤15s 能自然念完),其余卖点用**画面里的文字/字幕**呈现 | 台词变短,但这本就是信息流广告的打法(前 3 秒抓人) |
| B. 放开时长 | 允许 >15s(但 Seedance 上限 15s,且需求钉死 15s) | 与"只要15s"冲突,且 Seedance 本身不支持更长 |
| C. 接受口型漂移 | 全脚本硬塞 15s | 口型后半段乱、语速失真,广告质感差 |

方案默认走 **A**:节点里把"要口播的台词"限制在一个可配的字数上限(如 `max_spoken_chars≈40`),超出部分不进引号台词(可作为画面描述/屏幕字)。**这点要先和老板确认**——他要的是"全脚本都念出来"还是"一句钩子口播+画面带卖点字"。

---

## 3. 关键发现三:BGM 是「默认就有」,口型靠台词加引号

- **BGM**:Seedance 默认会自动配 BGM;要**压制**才在 prompt 写 `no music`。所以"配 BGM"= 不写 no music 即可(或在 prompt 描述想要的音乐风格)。
- **口播**:把台词用**双引号**括进 prompt 文本,模型就念出来并对口型;音色从人物描述推断,可在 prompt 显式写口音/语气(如"亲切、口语化")。
- **generate_audio**:布尔参数(部分平台默认 true),控制是否出声;开/关**计费一样**。我们显式传 `true`。

## 4. 改进方案:4 个增强(在 PR #48 基础上)

### 增强① 打开音频
provider 请求体加 `generate_audio: true`(profile `default_options.generate_audio` 可配,默认 true)。

### 增强② 广告口播 prompt 模板(复刻老板已验证的写法)
老板那套 prompt **已经被验证能出片**,直接对齐他的结构,别自己另发明:

```
请直出一条抖音信息流广告视频，配 BGM。
这是口播脚本。{<用户填的脚本>}
<角色名>出镜@<素材id>
```

节点做的事:把"用户脚本 + 选中的 @素材"拼成上面这个结构的文本,作为 `content` 的文本部分;`@<素材id>` 解析成真实参考图/视频放进 `content` 的 image_url/video_url。即对 Ark 实际发:

```
content = [
  { type:"text", text:"请直出一条抖音信息流广告视频，配 BGM。\n这是口播脚本。{海风小镇的街坊们…来逛逛！}\n老板娘出镜" },
  { type:"image_url", image_url:{ url:<老板娘参考图presign> }, role:"reference_image" }
]
ratio="9:16", resolution="720p", duration=15, generate_audio=true
```

- 模板做成**可编辑常量 + 占位符**(`{script}` / `{role}` / 指令前缀),先在节点内常量,后续可挪进 prompt-registry 给运营调。
- 指令前缀("直出抖音信息流广告，配 BGM")沿用老板措辞——它隐含了"画外音口播 + 自动字幕 + BGM + 多镜空镜"这一整套,Seedance 自己会编排,我们不用逐项描述。

### 增强③ 出镜人物/@素材引用
- 创作端打 `@` 选本 case 素材(见 §9),选中的素材 id 进 `reference_asset_ids`;节点把每个 id 解析→presign→放进 content 的 image_url(图)/video_url(视频)。
- prompt 文本里保留 `<角色>出镜` 的自然语言指代(对齐老板写法);**不强依赖** `[图N]` 占位符。
- ⚠️ **待确认(落地前对官方 doc)**:国内火山方舟(volces)`content` 数组里 **video_url/参考视频的精确字段名与 role 取值**、`generate_audio` 默认值、参考素材数量上限。官方:https://www.volcengine.com/docs/82379/1520757 。**文本 prompt 结构按老板已验证写法走,风险低;要核的是"参考素材怎么塞进 content"这部分的域内字段名。**

### 增强④ 视频参考支持
- 选图器 `IMAGE_KINDS` 扩展到也列**视频素材**(kind `video`/`portrait`);视频卡片复用现成 `TemplateAssetCard` 的 HTML5 预览。
- 节点把视频参考走火山 `reference_video`(provider 同样 presign 成公网链);区分图/视频两类 reference。
- 火山参考上限(国际版:9 图 + 3 视频)按域内 doc 核准后在节点/前端加校验。

## 9. 「AI素材」素材库 Tab 设计(本次新增需求)

目标:在后台**素材库**加一个 **「AI素材」Tab**,像现有视频素材那样**按 case 进入、上传素材**,Tab 内**分「图片」「视频」两类**(先只这两类,不复杂化);创作时能像老板的 prompt 那样 **@ 引用本 case 的 AI素材**。

### 数据层(零迁移,复用现成能力)
- AI素材就是 **MediaAsset**(kind=`image` 或 `video`),走现成上传链路 `/api/uploads/{prepare,file,complete}`(已支持传 `case_id`)。
- 用一个**标签 `tags=["ai_material"]`** 把它们和普通 broll/portrait 素材区分开(`MediaAssetRecord.tags` 字段已存在,**不用新表/新迁移/新 kind**)。
- 列表查询:`GET /api/media/assets?case_id=&kind=image|video`,前端再按 `tags 含 ai_material` 过滤(或给列表接口加个 `tag` 过滤参数,小改)。

### 素材库 UI(新 Tab)
- 在 library 页加 **「AI素材」** Tab(与 BGM/字体/音色/模板 Tab 并列)。
- 顶部**选 case**(复刻 `TemplatesTab` 的 case 选择器)。
- 选定 case 后,**两个分区:图片 / 视频**,各自网格展示 + 上传入口(图片走图片上传、视频走视频上传,视频自动出缩略图——`uploads.py` 已做)。
- 卡片复用现成 `TemplateAssetCard`(图片显缩略图、视频显缩略图+HTML5 预览)。

### 创作端 @ 引用
- 当前 `SeedanceReferencePicker` 已从持久库按 case 读素材;改为**只列本 case 的 `ai_material` 图片+视频**(把 `IMAGE_KINDS` 扩成 image+video,并加 ai_material 标签过滤)。
- 选中的素材 id 进 `reference_asset_ids`(已有字段),节点解析→(A 混剪:作为剪辑素材 / B 生成:presign 给 Seedance 当参考)。
- **可选体验**:提示词框加 `@` 弹素材库下拉(插入素材芯片),更贴近老板 `@老板娘` 的写法;功能与网格选择器等价,优先级低。

### 工作量小结
- 后端:列表加 `tag` 过滤(小);上传时打 `ai_material` 标签(小);其余复用。**零 DB 迁移**。
- 前端:新 Tab + case 选择 + 图片/视频两区 + 上传(中,主要是 UI);选图器扩到 image+video+tag 过滤(小)。
